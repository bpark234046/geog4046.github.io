Assignment 08: Insert a map into a web page

Create a web page or use a page from a previous assignment, and embed a map onto the page. You can use a data layer from a previous assignment, textbook example, or just display a basemap. Provide a way for users to toggle (show and hide) the map.

Process

Starting with a working web page, add the necessary HTML, CSS, and JavaScript to include a box displaying a map.

  1. Use a previous assignment or other examples to start with a basic web page with some content in the body. The page should reference separate JavaScript and CSS files—do not write JavaScript and CSS directly into the web page.
  2. Plan how the map will visually fit into the page and what CSS rules might be needed to achieve your design.
  3. Create a div to be the box that holds the map. It will need an id attribute.
  4. Add rules into your CSS files to style the map. Consider the height, in pixels [example], and whether the map should be hidden at first [example], requiring user input to show it, or if it will be visible initially and have an option to hide it.
  5. Refer to a previous assignment or textbook examples for the JavaScript code to create the map. The JavaScript code from the GTKWebGIS examples needs to be copied from between the script tags and placed into a separate script file (e.g., map.js). Do not copy unecessary code, such as the code to display two maps, code linking the navigation between two maps, and references to unused modules.
  6. Add a link or button the user can click to show or hide the map div. In a JavaScript file, include and modify a sample of code to listen for a click on the button or link and then toggle the map div element. [example]

Graded Requirements

  • Include jQuery JavaScript
  • Your own CSS
  • Your own JavaScript
  • Scripts and styles in separate files
  • Your own page title
  • Your own content
  • An interactive map with any layers or just a basemap
  • A control for the user to show and hide the map
  • Proper indentation
  • Commented JavaScript code
  • No errors in developer console
