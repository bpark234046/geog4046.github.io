Code Assignment 06d

Introduction to JavaScript - Interacting with the Document Object Model

Instead of typing portfolio entries directly into an HTML file, create empty HTML elements and let JavaScript fill in the information, such as title, link URL, image source, and description, by pulling the data from a JavaScript object.

Process

These steps assume you will copy code from the previous assignment and build upon it in a new .js file to add features to your portfolio.

  1. Create a new JavaScript file code-assignment-06d.js and include it with script tags in your portfolio page.
  2. Remove all project-specific text from the portfolio HTML and put the text into an object in your JavaScript file. See Code Assignment 6 for an example of how to structure your project metadata in a JavaScript object. Text to remove might include:
    • article id attribute values
    • a href attribute values
    • img src and alt attribute values
    • project title text
    • project description text

  3. Remove all but the first portfolio entry’s HTML. This leaves us with a blank template that we can clone and fill in for each portfolio item. For example:

     <article>
   <div class="left-column">
     <img src="" alt="">
   </div>
   <div class="right-column">
     <p></p>
   </div>
 </article>

    Keep the classes because they are not unique to an entry and should be applied to every entry. In this example they apply grid styles.

  4. In your portfolio HTML, create a unique id attribute for:
    • the parent HTML element that contains your portfolio entries, e.g., main (the id allows JavaScript to find this element, save it to a variable, and add new portfolio entries to it)
    • the first element that constitutes a portfolio entry, e.g., article. This element will hereafter be refered to as the “portfolio entry element”.
  5. In your JavaScript file, create three functions or class methods to: 1) copy the template HTML, 2) fill the HTML with data, and 3) write the HTML to the page.
    • The first function/method will need to get the portfolio entry element by its id, clone it (and its children), set a new id on the clone, and return the cloned element.
    • The second function/method should get the portfolio entry element’s children and give them new attribute and innerHTML values. These values are pulled from the project data. The function/method returns the portfolio entry element.
    • The third function/method can then loop through the project data, and for each portfolio entry, write a portfolio entry element to the page.
