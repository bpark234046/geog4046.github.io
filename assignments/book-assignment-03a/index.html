Book Assignment 03a

Crowdsourced GIS in ArcGIS Online

Complete Assignment 3A in Chapter 3 of GTKWebGIS to create a “volunteered geographic information system” where the public can submit input and the crowdsourced data will appear on the map.

Process

Review the tutorial and assignment in the book for a detailed explanation of the process. Below is an overview of the steps involved.

  1. Prepare a CSV with sample data to act as a template. The attributes are given in the book.
  2. Publish and share a hosted feature layer from your CSV
  3. Define feature templates from your hosted feature layer with appropriate icons
  4. Create a web map centered on LSU containing your hosted feature layer
  5. Create a web app from the map so the public can add data to your hosted feature layer
  6. Configure the app
  7. Share the app
  8. Test the app’s accessibility in incognito/private browsing mode
  9. Add a link, screenshot, and description of the app in your portfolio

Graded Requirements

  1. An ArcGIS Online web app showing a hosted feature layer with feature templates for the four suspects, and two sample “sightings” that you input
  2. A form to collect new input from the public, including photo uploads
  3. A map centered on LSU displaying the input features
  4. Popup for each input feature, showing the feature’s relevant attributes
  5. Customizations to give your app a professional look and feel
  6. A screenshot, description, and link to your web app on your portfolio website

Submit on Moodle

  1. The REST URL of your feature layer
  2. The URL of your web app
