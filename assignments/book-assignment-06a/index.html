Book Assignment 06a

Publishing a web service to ArcGIS Server

Complete Assignment 6A in Chapter 6 of GTKWebGIS, using the chapter tutorial and Q&A as guidance, to publish the book data as a map service from ArcMap to our class ArcGIS Server, and then add the service to a web map/app with a time slider to animate the data.

Process

Review the tutorial and assignment in the book for a detailed explanation of the process. Below is an overview of the steps involved.

  1. Connect to the class GIS server using the connection file provided on Moodle (skip GTKWebGIS section 6.1 and use the connection file instead). Download and unzip copy-ags.zip from Moodle. Double-click copy-ags.cmd. This should copy publisher4046@webgis.ga.lsu.ags to your profile. Restart ArcMap. The server connection should now appear under GIS Servers in the Catalog Pane. If needed, the file can be manually copied to %APPDATA%\ESRI\Desktop10.5\ArcCatalog.
  2. Use ArcMap to create a map using the book data for Chapter 6, setting appropriate symbology. Section 6.2 demonstrates this step in detail and there are tips on textbook page 243.
  3. Enable time on your data layer(s), explained in section 6.3.
  4. Publish your map as a service in your own folder on the class GIS server (see 6.4).
    a. Create your own folder at step 4 with your user name, which could be the same as your Github or ArcGIS Online/Portal:

    b. Leave the default service options but enter info into the item description fields:

    c. Click Analyze. If you get an error about time not being enabled, open the time slider and click disable time and then click enable time:

    d. View your published server in the ArcGIS REST Services Directory on the class GIS server (6.5): https://webgis.ga.lsu.edu/arcgis
    e. Copy the service URL, which should look like:
    https://webgis.ga.lsu.edu/arcgis/rest/services/username/example_service_name/MapServer/0
  5. On a new map in ArcGIS Online, choose “Add Layer from Web” and paste your service URL (6.6).
  6. Enable popups showing relevant attribute data (see Book 2).
  7. Share the map and create an app using either Web AppBuilder or a configurable app.
  8. In the app configuration, use time settings that will animate the features at a reasonable speed.
  9. Customize, brand, share, and test the app.

Graded Requirements

  1. An ArcGIS Online web app showing the map service you published from ArcMap
  2. The layer must be time-enabled so the app can animate the data
  3. Customizations to give your app a professional look and feel

Submit on Moodle

  1. The URL of your web app
  2. The REST URL of your map service
