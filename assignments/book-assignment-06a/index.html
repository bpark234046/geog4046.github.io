Book Assignment 06a

Publishing a web service to ArcGIS Server

Complete Assignment 6A in Chapter 6 of GTKWebGIS, using the chapter tutorial and Q&A as guidance, to publish the book data as a map service from ArcMap to our class ArcGIS Server, and then add the service to a web map/app with a time slider to animate the data.

Process

Review the tutorial and assignment in the book for a detailed explanation of the process. Below is an overview of the steps involved.

  1. Connect to the class GIS server using the connection file provided on Moodle (skip GTKWebGIS section 6.1).
  2. Use ArcMap to create a map using the book data for Chapter 6, setting appropriate symbology.
  3. Enable time on your data layer(s).
  4. Publish your map as a service in your own folder on the class GIS server.
  5. Add your map service to a new map in ArcGIS Online.
  6. Enable popups showing relevant attribute data.
  7. Configure the time settings so the map can be animated at a reasonable speed.
  8. Customize, brand, share, and test the app.
  9. Add a link, screenshot, and description of the app in your portfolio

Graded Requirements

  1. An ArcGIS Online web app showing the map service you published from ArcMap
  2. The layer must be time-enabled so the app can animate the data
  3. Customizations to give your app a professional look and feel
  4. A screenshot, description, and link to your web app on your portfolio website

Submit on Moodle

  1. The URL of your web app
  2. The REST URL of your map service

Tutorial Notes

  • How to connect to the class ArcGIS Server: Download and unzip copy-ags.zip from Moodle. Double-click copy-ags.cmd. This should copy publisher4046@webgis.ga.lsu.ags to your profile. Restart ArcMap. The server connection should now appear under GIS Servers in the Catalog Pane. If needed, the file can be manually copied to %APPDATA%\ESRI\Desktop10.5\ArcCatalog.
