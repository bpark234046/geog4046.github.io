Code Assignment 08

jQuery - Ajax and JSON

In previous assignments, we wrote our portfolio metadata directly into a JavaScript file and saved it to a variable, an array of objects. In this assignment, we will store our data in a separate “JSON” file and retrieve the contents from our script using Ajax.

Process

  1. Create a new JavaScript file code-assignment-08.js based on the previous assignment and include it with script tags in your portfolio page.
  2. Create a new .json file and save it in the same folder as your .js file.
  3. Cut the portfolio data array from the previous assignment and paste it into the new JSON file.
  4. Modify the portfolio data array to conform to JSON formatting:
    • Remove variable declarations, e.g., projectData[0] = , leaving you only with objects. It should look something like this: 
       {
   id: 1,
   title: 'title of the first assignment'
 }
 {
   id: 2,
   title: 'title of the second assignment'
 }
    • Enclose these objects in [] because our original projectData variable was an array containing all of the portfolio objects.
    • Separate the objects with a comma, since they are different elements in the array.
    • Enclose the property names in double quotes, ".
    • Change the strings’ single quotes to double quotes.
    • Save the file. It should look something like this: 
       [
 {
   "id": 1,
   "title": "title of the first assignment"
 },
 {
   "id": 2,
   "title": "title of the second assignment"
 }
 ]
    • View the .json file in a browser, e.g., https://example.github.io/data/projectData.json.
  5. At this point, the code copied from the previous assignment references a projectData variable that held the project metadata, but we removed the declaration in the previous step. Use Ajax to get the project data and save it to a new projectData variable.
    • To implement Ajax, use jQuery’s getJSON method. It takes two arguments as input, the .json file URL and a function.
    • The function should have one argument. The variable name should reflect that it contains the JSON data.
    • The contents of the function should be the actions that need to be performed with the data, such as creating a new Project object and calling a method to create portfolio entries on the page. It should not contain the init function or the page load event handler.

This sample code shows the getJSON method’s syntax and arguments: the URL variable and the callback function:

 let url = 'https://example.github.io/data/projectData.json'
 function createPortfolio (jsonData) {
   // code to do something with the JSON data goes here.
 }
 $.getJSON(url, createPortfolio)

The URL value and the function can be passed as arguments directly instead of creating variables:

 $.getJSON('https://example.github.io/data/projectData.json', function (jsonData) {
   // code to do something with the JSON data goes here.
 })
