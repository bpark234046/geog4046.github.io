Final Project

A Leaflet Map

Create a web map using Leaflet, an open-source JavaScript library. Map the dataset of your choice and customize the symbology and popups to convey meaningful information to your users.

Data

Find a dataset or create one yourself. It should be related to a topic you are interested in. Choose something that would be fun/useful to map and could potentially be used outside of this class.

Process

  1. Create a new folder in your repository, giving it a name appropriate for the theme of the map rather than something generic like “final”. Keep in mind, the web address will reflect this name, e.g., https://yourname.github.io/foldername. Also create index.html, a .css file, and a .js file inside the folder.
  2. In index.html, create a basic HTML skeleton and add elements and CSS rules to give it a layout and style, or use a Bootstrap template.
  3. Create a map div in the .html file and in the .js file add code for a basic Leaflet map.
  4. Use additional L.tileLayer methods to add basemaps.
  5. Use the L.control.layers method to create a control where the users can switch basemaps. See this example code to include a GeoJSON layer in the control.
  6. Prepare your data. If using an existing dataset that is not already in GeoJSON format, it can be converted using an online converter, or the desktop GIS application QGIS. If creating your own dataset, you can either create a GeoJSON file (try using geojson.io) or use Leaflet’s vector drawing methods.
  7. Add your data to the map:
    a. For GeoJSON, create a variable for the GeoJSON object and pass it to the L.geoJSON method.
    b. For other data, use methods like L.polygon or L.polyline to draw features on the map.
  8. Apply custom styles or icons to the features. If you have a point layer in GeoJSON look at these code samples to customize the marker:
    a. Custom icon for GeoJSON [code]
    b. Circle marker for GeoJSON [code ]
  9. Use the bindPopup method to create a popup. The L.geoJSON method has the onEachFeature option that can be used to create a popup on each feature and use the feature’s attribute values in the popup.
  10. Depending on the complexity of the symbology, a map legend might be appropriate. Otherwise, a description could simply be included in the webpage.
  11. Update your portfolio to include all of the webmaps in this class that are still functioning (maps made with an expired ArcGIS Online trial can be removed)

Graded Requirements

  1. A web page with valid HTML hosted online
  2. Page title; no errors in console; scripts and styles in separate files; indentation
  3. A visible Leaflet map
  4. At least three basemaps the user can switch between, or just one custom basemap
  5. At least one vector layer (e.g., point, line, or polygon) created either with Leaflet’s vector drawing methods or GeoJSON.
    a. Must be unique and cannot be used by anyone else in the class (see the reservation sheet on Moodle).
    b. Must have at least five features
  6. Non-default icon or symbology applied to the vector layer(s)
  7. A popup on a vector layer
    a. Use of at least one attribute of the clicked feature
    b. Use of HTML in the popup
  8. Some kind of logic that changes some aspect of the map based on an attribute of the data, e.g., symbol color, custom icon, or popup content.
  9. A legend or explanation of the map contents
  10. Professionalism: the web page containing the map should have customizations like style and layout appropriate for your project, and the map itself should function well.
  11. An updated portfolio
