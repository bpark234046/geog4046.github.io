Book Assignment 05

Build your own app with Web AppBuilder in ArcGIS Online

Create a new web app with ArcGIS Online Web AppBuilder using your own data, either a new dataset or a feature layer published in a previous assignment.

Process

Do not directly follow the assignment in chapter 5 for this exercise. Instead, refer to the chapter as a rough guide for completing the steps below. Some widgets will not work with Portal accounts.

  1. Research and download a dataset (look for CSV format or zipped shapefiles) with multiple attributes that can be used with widgets (see examples in textbook 5.4). For data, see the class Portal group (download the zip files, add them to your map), the data sources page, or use any other data you have. Layers can be used with the required widgets if they have attributes that can be charted (magnitude, population, etc.), filtered (by age, date, size, etc.), queried (“all points within 1 mile”), statistically summarized, or time-enabled (has a date field).
  2. In ArcGIS Online/Portal, under “Content”, add your downloaded dataset’s csv, zip, etc., choosing to publish it as a hosted feature layer.
  3. Also under Content, create a new web map and add your layer(s) to it.

  4. Save and share the map, choosing to create a web app with Web AppBuilder
  5. In Web AppBuilder, add and configure data-dependent widgets–those that interact with the map layer data, such as a chart, filter, group filter, infographic, query, related table charts, summary, or time slider. There are other widgets but not all will function with Portal accounts.
  6. In Web AppBuilder, use the options to customize, brand, share, and test the app.

Graded Requirements

  1. An ArcGIS Online web app created with Web AppBuilder
  2. Data different from the tutorial
  3. At least four widgets; choose from: chart, filter, group filter, infographic, query, related table charts, summary, time slider
  4. Customizations to give your app a professional look and feel
