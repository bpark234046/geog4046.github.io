Midterm Project

Apps to view and analyze data

You are the Founder, President, and CEO of FFS Consulting, LLC. You received an email from a potential client who describes some requirements for a project. Find your own data to use as the client’s data, or choose from the examples below, and create the maps, apps, and web services to meet the client’s requirements.

From: Rusty Shackleford
Sent: Monday, February 26, 2018 7:00 AM
To: FFS Consulting
Subject: Web mapping project

Dear Sir or Madam,

I was referred to you for your web mapping services. My company has some data layers that we work with and share with clients. Currently, we do some analysis and stuff with ArcMap and then share the maps with clients as printouts or PDFs. We want to get something set up online, though, so we can send our clients links to web pages with maps on them instead of PDFs. Also, if possible, we want our analysts to be able to work on the data online instead of ArcMap. So I’m thinking we could have one web page for our internal use where our folks can analyze a dataset, and then a separate web page we share with clients where they can view layers with intuitive symbology. We want the clients to have some flexibility with how they use the online map, like filtering, querying, and summarizing the data, and maybe some additional visualizations like popups and charts. It would be great if they could even edit a layer themselves right there in the web map. I’m open to your input on what other kinds of bells and whistles could make the map better.

If that sounds like a project you’re interested in, could you send me your portfolio so I have some idea what to expect? Thanks in advance, and I look forward to seeing what’s possible with web GIS!

Best regards,

Rusty Shackleford

Process

  1. Create a new ArcGIS Online trial account if your trial will expire soon, or use your class Portal account.
  2. Gather your data and, if necessary, add to “My Content” as a hosted feature layer.
  3. Create a map. This map can potentially be shared by both the “viewer” and “analysis” apps. Another option is to make two maps, one for each app.
  4. Configure symbology, popups, default view, etc.
  5. Share the map(s) and create the analysis app first with Web AppBuilder.
  6. Follow Book 7a to create the analysis app with the analysis widget.
  7. Create the viewer app with Web AppBuilder and add the required widgets and customizations.
  8. Update your portfolio to have at least four entries and make sure the HTML and CSS are valid with proper indentation.

Graded Requirements

For quantities of layers, widgets, etc., consider the required numbers a minimum. You can always have additional layers and widgets.

  1. One ArcGIS Online web map
  2. Two web apps (one “analysis app” and one “viewer app”)
  3. For the analysis app:
    a. Two tools in the analysis widget that will work with a layer (ch 7)
    b. One layer that will work with the two tools above (ch 7)
    c. Customizations to give the app a professional look and feel
  4. For the viewer app:
    a. Three data-dependent widgets (ch 5)
    b. Three other non-default widgets (ch 5)
    c. One layer that will work with the data-dependent widgets (ch 5)
    d. One layer added from the web (e.g., REST URL; see ch 6)
    e. One editable hosted feature layer (ch 3)
    f. Popup enabled on one layer with some kind of customization, such as changing the field alias (ch 2)
    g. Custom symbology on all layers where possible (ch 2)
    c. Customizations to give the app a professional look and feel
  5. For the portfolio:
    a. Valid HTML and CSS hosted online (JavaScript not required)
    b. At least four entries with a screenshot, link, and description
    c. Customizations such as a logo, font, or colors that distinguish your portfolio
  6. Take screenshot(s) of your analysis widget’s options before running the tools, and a screenshot of the output, such as a table or a new layer drawn on the map.
