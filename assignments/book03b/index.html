Book Assignment 3B

Crowdsourced GIS in ArcGIS Online

Complete Assignment 3B in Chapter 3 of GTKWebGIS to create a “volunteered geographic information system” where the public can submit input and the crowdsourced data will appear on the map.

Process

Review the tutorial and assignment in the book for a detailed explanation of the process. Below is an overview of the steps involved.

  1. Get the book’s assignment 3 data (file geodatabase and zipped shapefiles)
  2. Publish and share hosted feature layers from either the file geodatabase or shapefiles.
  3. Come up with three incident types (for point, line, and polygon) and define a feature template for each type from your hosted feature layers, using appropriate icons
  4. Create a web map containing your hosted feature layers
  5. Create a web app from the map so the public can add data to your hosted feature layer
  6. Configure the app
  7. Share the app
  8. Test the app’s accessibility in incognito/private browsing mode
  9. Add a link, screenshot, and description of the app in your portfolio

Graded Requirements

  1. An ArcGIS Online web app showing a hosted feature layer with at least two feature templates, and two sample records already input
  2. A form to collect new input from the public, including photo uploads
  3. A map of your chosen area displaying the input features
  4. Popup for each input feature, showing the feature’s relevant attributes
  5. Customizations to give your app a professional look and feel
  6. A screenshot, description, and link to your web app on your portfolio website

Submit on Moodle

  1. The REST URLs of your feature layers
  2. The URL of your web app
