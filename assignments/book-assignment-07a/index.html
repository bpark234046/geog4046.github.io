Book Assignment 07a

Analysis tools in ArcGIS Online

Complete Assignment 7A in Chapter 7 of GTKWebGIS, using the chapter tutorial and Q&A as guidance, to create a web app capable of processing spatial data using the analysis widget in ArcGIS Online’s Web AppBuilder. You may choose any theme and corresponding data, as long as the data can be analyzed with the tools available in the Web AppBuilder “Analyze” widget.

Process

Review the tutorial and assignment in the book for a detailed explanation of the process. Below is an overview of the steps involved.

  1. Find data to display and analyze on a map.

    When searching for a dataset, keep in mind that you will need to employ the analysis widget, and specifically use data-driven analysis tools, meaning you need to find data compatible with the widget. For example, the “summarize within” tool can count the points that fall inside each polygon in a polygon layer, so a compatible dataset would be a point layer of traffic accidents and polygon layer of zip codes. The tool could count how many accidents per zip code. See the Tutorial Notes section to familiarize yourself with some data-driven tools available in the analysis widget.

    You can look for ArcGIS Hub (formerly ArcGIS Open Data) sites for zipped shapefiles or just REST service URLs. Also see the class Portal group.

  2. Add your data to “My Content” as a hosted feature layer if you want all analysis tools to be work. Certain tools will also work with a layer added from the web (e.g., an ArcGIS Server REST URL). With ArcGIS Online (but not Portal), you can also add related data from Living Atlas.
  3. Create a new map. Add your data from various sources and change their symbology. Note: it is not necessary to click Save Layer on a hosted feature layer after changing its symbology. Doing so will save the symbols with the layer itself, not the map. This means it will keep the same symbology if you add it to a new map.
  4. Create a new app with Web AppBuilder and add the analysis widget.

  5. Choose tools from the analysis widget in Web AppBuilder to perform some kind of useful analysis of your layers. All tools are available with an ArcGIS Online trial account, but Portal users should read the Tutorial Notes for a list of analyses that work in the class Portal.

    Always uncheck the box Save the result in user's account for class assignments.

  6. Take screenshots of your analysis widget options before running the tools, and screenshot the output. Screenshots are necessary because the analysis widget is not publicly accessible; when someone opens the widget they will be prompted to sign into your ArcGIS Online organization.
     
  7. Customize, brand, share, and test the app.

Graded Requirements

  1. An ArcGIS Online web app using the analysis widget
  2. At least two data-driven tools in the analysis widget
  3. At least one data layer that can be analyzed with the tools above
  4. Customizations to give your app a professional look and feel such as a logo, a descriptive app title, a subtitle, links to other apps or sites, different layouts, or a color scheme.
  5. Take screenshot(s) of your analysis widget’s options before running the tools, and a screenshot of the output, such as a table or a new layer drawn on the map.
