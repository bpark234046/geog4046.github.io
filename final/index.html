Final Project: JavaScript Web App

Use Leaflet and the ArcGIS JavaScript API to create an application displaying interactive maps. Each map should have layers that make sense, and the application should have some controls that change the map(s) or map view(s).

Process

You will need to create an HTML page to hold the maps and interface elements; CSS to style the HTML, especially the map containers; and JavaScript to display the maps and layers, and handle user input.

  1. Create files and folders
  2. Start with a basic HTML page that links all necessary libraries: CSS and JavaScript for Leaflet, ArcGIS, jQuery, and Bootstrap. Your site can be a single HTML file (index.html) or multiple HTML files (e.g., index.html, map.html, about.html); if you create multiple pages, designate one as the homepage with links to the other pages.
  3. Add HTML and CSS to create interface elements, such as a navigation bar, input elements, boxes of content, or buttons. Use Bootstrap to help build your interface. The grid system might come in handy.
  4. Give the app a "brand"—consistent styling, organization name, and a generally professional appearance that makes the page look like a functional product and not a technical demo. Add text, images, or other media to help fill out the page(s) with content.
  5. Add a Leaflet map to your page, or create a separate page linked from your home page. See the Leaflet quick start example to create a basic map.
    • Add a basemap (tile layer), covered in the quick start guide at "Next we'll add a tile layer...". See this list of providers for additional basemaps.
    • Add GeoJSON layers to the map. You can find data already in GeoJSON format on ArcGIS Open Data (in the APIs menu on the dataset page) and you can convert shapefiles to GeoJSON using the free QGIS. Pick from these BR datasets or find your own:
      BREC Parks,
      BR Sports Venues,
      BR Recreation,
      BR Libraries
      You may not use earthquake data.
    • Create a popup or some other display of information for each feature. See the onEachFeature section of Leaflet's GeoJSON documentation. [example]
    • Apply custom styles or markers where needed to create a readable, informative map. See the pointToLayer section of Leaflet's GeoJSON documentation and this [example]. If you want to design your own icon, see the Leaflet custom icons example (stick to the "Creating an icon" section) and refer to this [example]
    • Add controls to show/hide layers.
      • You can use Leaflet's built-in layer controls and see the easy basemap switcher [example] or the more difficult GeoJSON layer toggler [example].
      • You could also make your own buttons in HTML with similar functionality. For example, create a button in HTML, listen for clicks on it with jQuery, and run a Leaflet function to toggle a layer when clicked. [example]
    • Use a plugin to add extra functionality to Leaflet. See this list of plugins.
      • Read your chosen plugin's documentation and look for a working demo.
      • Plugins will typically require you to download a .js file to put into your js folder. You then link to the file from your HTML, the same way you link to other JavaScript files.
      • Some plugins might also require you to download and link to a CSS file.
      • Plugin CSS and JS files should go after (below) the Leaflet CSS/JS but before (above) your own CSS and JS files in the head and body, respectively.
      • Finally, you'll probably need to add some JavaScript to call the plugin's functions.
  6. Add some JavaScript to create a map with the ArcGIS JavaScript API.
    • ArcGIS and Leaflet code can coexist in the same file or separate files, just remember to link to both in your HTML.
    • The map can either be on the same page as your Leaflet map, or on a separate page.
    • You can use the 2D or 3D code from the 2d3d_Views_key.html textbook example, or something else.
    • You can create 3D scenes with a free ArcGIS Online account and add them to your site (optional). [example]
    • You can publish a map as a web service on the class GIS server (optional).
    • If you created HTML map controls for Leaflet, you could make them work for the ArcGIS map, too, using equivalent functions, e.g., Leafet maps have flyTo() and ArcGIS web scenes have goTo() (optional) [example]
    • See GTKWebGIS chapters 6 (publishing), 9 (creating 3D scenes), and 10 (JavaScript API).
    • The controls you create for Leaflet don't have to work with your ArcGIS map.
  7. Collect user input, for example using a text box, checkbox, or select dropdown. Use JavaScript to respond to the user's input, such as manipulating the page or map.

Graded Requirements

  • Scripts and styles in separate files
  • Include jQuery JavaScript
  • Include Bootstrap CSS and JavaScript
  • Include mapping library CSS and JavaScript
  • Your own CSS rules
  • Your own JavaScript code
  • Your own page title
  • Your own content and brand
  • Use of Bootstrap (any CSS or JavaScript component)
  • A Leaflet map
    • Basemap
    • Operational layers from GeoJSON (at least 2)
    • Popups or similar display of layer attributes for GeoJSON layers
    • Custom icons or styles for GeoJSON
    • Controls to show/hide layers
    • Leaflet plugin (at least 1)
  • An ArcGIS map
    • Basemap
    • Operational layer from any source (at least 1)
  • Allow user input or interaction (outside of the map)
  • Respond to user input (e.g., display their input, change the map view)
  • Proper indentation
  • Commented JavaScript code
  • No initial errors in developer console
